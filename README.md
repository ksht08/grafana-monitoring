# Система мониторинга тестового приложения

Данный репозиторий содержит конфигурацию для развертывания стенда мониторинга на базе Grafana, Prometheus и Loki. Проект предназначен для отслеживания технического состояния микросервисов и анализа бизнес-метрик.

## Адреса сервисов

После запуска проекта сервисы доступны по следующим адресам:

* **Grafana**: http://localhost:3000 (Интерфейс мониторинга)
* **Тестовое приложение**: http://localhost:5000 (Объект мониторинга)
* **Prometheus**: http://localhost:9090 (База данных метрик)

## Структура Dashboards

Дашборд в Grafana разделен на три ключевых сегмента для комплексного анализа.

### 1. Анализ логов (Logging)
Визуализация текстовых данных из приложения. Позволяет быстро оценить стабильность системы по категориям:
* **INFO**: Информационные сообщения о штатной работе системы.
* **WARNING**: Предупреждения о некритичных отклонениях.
* **ERROR**: Ошибки, требующие анализа и вмешательства.

### 2. HTTP Метрики
Мониторинг сетевой активности и качества обработки запросов. Панели отображают распределение по HTTP-кодам:
* **2xx**: Успешные запросы.
* **4xx**: Ошибки на стороне клиента (неверные запросы, отсутствие прав).
* **5xx**: Ошибки на стороне сервера (сбои логики приложения).

### 3. Бизнес-события (Business Events)
Специализированный раздел для отслеживания пользовательского опыта и ценных действий:
* Количество созданных заказов.
* Успешные регистрации пользователей.
* Активность в целевых воронках приложения.

## Запуск проекта

Для развертывания стенда необходимо наличие Docker и Docker Compose. Все зависимости и связи между контейнерами преднастроены.

Для запуска выполнить:
    ```bash
    docker-compose up -d
    ```
